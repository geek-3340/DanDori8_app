# React + TypeScript + Laravel + PostgreSQL + Docker 環境構築手順

前提：Windows11 + WSL2のLinux仮想環境下

## 1. PHP/Composerをインストール

```bash
/bin/bash -c "$(curl -fsSL https://php.new/install/linux/8.5)"
```

## 2. Laravelインストーラーをインストール

```bash
composer global require laravel/installer
```

## 3. スターターキットでLaravelアプリを作成

```bash
laravel new my-app
```

## 4. Laravel Sailをインストール

**Laravel Sailとは？**
Laravelアプリの開発環境を手軽にDocker化できる軽量なコマンドラインインターフェイス

作成したLaravelアプリに移動
```bash
cd ~/my-app
```
Laravel Sailをインストール
```bash
composer require laravel/sail --dev
php artisan sail:install

# 拡張機能を選択 (PostgreSQLを選ぶ)
┌ Which services would you like to install? ───────────────────┐
│ pgsql                                                        │
└──────────────────────────────────────────────────────────────┘
```

## 5. PostgreSQLの設定

Laravelアプリはデフォルトで**MySQLとRedisの設定がある**ので削除する

`compose.yaml`を開いて、以下のように修正
```yaml
services:
  laravel.test:
      # 省略
      depends_on:
          - pgsql
          - mysql # 削除
          - redis # 削除
# 省略
volumes:
    sail-pgsql:
        driver: local
    sail-mysql:       # 削除
        driver: local # 削除
    sail-redis:       # 削除
        driver: local # 削除
```

`.env`を開いて、以下のように修正
>`.env.example`も`.env`と同様に修正
```bash
# 省略
DB_CONNECTION=mysql # pgsqlへ書き換え
DB_HOST=mysql       # pgsqlへ書き換え
DB_PORT=3306        # 5432へ書き換え
# 省略
```

`database.php`を開いて、以下のような`pgsql`以外の設定を削除する
>`mysql`や`sqlite`など！
```php
'connections' => [
        'pgsql' => [
            'driver' => 'pgsql',
            #省略
        ],
    ],
```

## 6. Laravelアプリを立ち上げる

Dockerコンテナの起動
```bash
./vendor/bin/sail up -d
```

フロントエンドの依存関係をインストール
```bash
./vendor/bin/sail npm install
./vendor/bin/sail npm run build
```

PostgreSQLのDBをマイグレーションして初期化する
```bash
./vendor/bin/sail artisan migrate
```

アプリの立ち上げ成功
```
http://localhost　にアクセスする
```
